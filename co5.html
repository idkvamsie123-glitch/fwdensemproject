🔌 What is a REST API?

A REST (Representational State Transfer) API is a way for two systems to communicate over HTTP. APIs expose endpoints (URLs) that accept requests and return data, usually in JSON format.

Base URL: https://api.example.com

GET    /users          → Get all users
GET    /users/42       → Get user with id=42
POST   /users          → Create a new user
PUT    /users/42       → Update user 42 (full)
PATCH  /users/42       → Update user 42 (partial)
DELETE /users/42       → Delete user 42

📦 JSON – JavaScript Object Notation

JSON is the standard data format for APIs. It's human-readable, language-independent text that represents structured data. Use JSON.parse() to convert a JSON string to a JS object and JSON.stringify() to convert back.

// JSON string (from API)
'{"name":"Alice","age":25,"skills":["HTML","CSS"]}'

// Parse JSON string → JS Object
const user = JSON.parse(jsonString);
console.log(user.name);   // "Alice"

// Stringify JS Object → JSON string
const json = JSON.stringify(user, null, 2);
// null = no replacer, 2 = indent spaces

🌐 Fetch API

The Fetch API is the modern way to make HTTP requests in JavaScript. It returns a Promise, so you use .then() chains or async/await. Always handle errors with try/catch or .catch().

// GET request
async function getUsers() {
  const res  = await fetch('https://jsonplaceholder.typicode.com/users');
  const data = await res.json();
  console.log(data);
}

// POST request (send data)
async function createUser(user) {
  const res = await fetch('/api/users', {
    method: 'POST',
    headers: { 'Content-Type': 'application/json' },
    body: JSON.stringify(user)
  });
  return await res.json();
}

💾 localStorage & sessionStorage

Web Storage APIs let you persist data in the browser. localStorage persists even after the browser closes, while sessionStorage clears when the tab closes. Both store data as strings.

// localStorage — persists after browser close
localStorage.setItem('theme', 'dark');
const theme = localStorage.getItem('theme');
localStorage.removeItem('theme');
localStorage.clear();

// Store objects (must stringify)
const user = { name: 'Alice', age: 25 };
localStorage.setItem('user', JSON.stringify(user));
const saved = JSON.parse(localStorage.getItem('user'));

🚀 Deploying Your Website

Modern web deployment is simple and often free. Popular options include GitHub Pages (static sites), Netlify (drag and drop or Git integration), and Vercel (optimized for JS frameworks).

# GitHub Pages (free, public repos)
git init
git add .
git commit -m "Initial commit"
git push origin main
# → Enable Pages in repo Settings

# Netlify CLI
npm install -g netlify-cli
netlify deploy --dir=. --prod

# Vercel CLI
npm install -g vercel
vercel --prod

🔒 CORS & API Keys

CORS (Cross-Origin Resource Sharing) controls which domains can access an API. API keys authenticate your requests. Never expose API keys in client-side code — use a backend proxy instead.

// ❌ NEVER expose API keys in frontend code
const API_KEY = "secret-key-12345";  // BAD!

// ✅ Use environment variables (backend)
const API_KEY = process.env.API_KEY;

// CORS error: server must allow your origin
// Response header needed from server:
// Access-Control-Allow-Origin: *
// or
// Access-Control-Allow-Origin: https://yoursite.com
