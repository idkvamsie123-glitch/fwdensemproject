Internet & Web Basics

Understand how the internet works, HTTP/HTTPS protocols, DNS resolution, browsers, and the client-server model.

HTTPDNSProtocolsBrowsers

📡 What is the Internet?

The Internet is a global network of computers connected using standardized communication protocols. It was originally developed as ARPANET in the late 1960s. Data travels in small packets through routers and switches across physical cables, fiber optics, and wireless signals.

User → ISP → Backbone Network → Destination Server
Data travels as packets, reassembled at destination.

🌐 HTTP vs HTTPS

HTTP (HyperText Transfer Protocol) is the foundation of data communication on the web. HTTPS adds TLS/SSL encryption for secure data transfer. Always prefer HTTPS for any website that handles user data.

HTTP  → Port 80  → Unencrypted (plain text)
HTTPS → Port 443 → Encrypted with TLS/SSL

GET /index.html HTTP/1.1
Host: www.example.com
Accept: text/html

🔍 DNS – Domain Name System

DNS translates human-readable domain names (like google.com) into IP addresses (like 142.250.67.46) that computers use to identify each other. It acts like a phonebook for the internet.

Browser → DNS Resolver → Root Server → TLD Server
       → Authoritative Server → Returns IP Address
       → Browser connects to IP

🔄 Client-Server Model

The web operates on a client-server architecture. The client (browser) sends requests, and the server responds with resources. REST APIs follow this model using HTTP verbs.

Client (Browser)          Server
    |                         |
    |--- GET /page.html ----→ |
    |                         |
    |←-- 200 OK + HTML -----  |
    |                         |
    |--- POST /login -------→ |
    |←-- 302 Redirect ------  |

📦 HTTP Status Codes

Servers communicate the result of a request using status codes. Understanding these is essential for debugging web applications.

1xx → Informational   (100 Continue)
2xx → Success         (200 OK, 201 Created)
3xx → Redirection     (301 Moved, 302 Found)
4xx → Client Errors   (404 Not Found, 403 Forbidden)
5xx → Server Errors   (500 Internal Server Error)

🏗️ How a Browser Works

When you type a URL, the browser: resolves DNS, opens a TCP connection, sends an HTTP request, receives a response, then parses HTML → builds DOM → applies CSS → executes JS → renders the page.

URL Typed
  → DNS Lookup (IP resolved)
  → TCP Handshake (3-way)
  → HTTP Request sent
  → Server responds with HTML
  → Browser parses DOM
  → Loads CSS, JS, Images
  → Page is rendered
